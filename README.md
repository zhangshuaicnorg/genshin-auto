# 该程序需要JRE25.0+



# 1.文件介绍

#### 1.sheet

存放曲目

#### 2.auto_save.txt

每次关闭程序自动保存内容

#### 3.config.json

配置文件

#### 4.format.xml

所支持的曲目格式，支持自定义和手动处理



# 2.更新简介

### v1.0.1

初版

**功能总览**

> 支持rollback
>
> 支持手动演奏(F11)
>
> 支持自定义格式以及处理脚本(默认支持 [.txt] [.dms])
>
> 自动保存
>
> 可以修改播放速率系数
>
> 修复了最小tick值计算错误的问题
>
> 添加了进度条可以调整播放进度
>
> 可能出现的bmp容错兼容



### v1.0.2

> 优化了进度条功能，调节进度条时自动扫描该进度的bpm值
>
> 优化了解析问题，自动排除误识别的无意义音符
>
> 修复了最小tick值计算错误的问题
>
> 修复了bpm识别逻辑错误 && 改成 ||
>
> 支持了琶音
>
> config.json中添加了arpeggio_delay_per(琶音延迟系数，每个琶音间隔用beat乘以这个数)



### v1.0.3

> 全面升级jcmd的语法和执行逻辑
>
> > 可以直接使用中缀运算
> >
> > 函数定义的逻辑升级与纠正
> >
> > 可以使用plugin
>
> 自动忽略[1-9]=[a-zA-z]，即自动忽略演奏音调的设定，默认为游戏能够接受的C大调
>
> 支持bpm，rollback关键字染色



### v1.0.4

> 全面升级进度表现方式
>
> 添加一块顶层面板显示，精确到每个音符
>
> > alt + F12可以隐藏面板
> >
> > 滚轮可以调节面板透明的
>
> format文件中添加两个模块
>
> <auto_save>和<plugin>
>
> > <auto_save>中的read和write是用来管理自动保存和读取的
> >
> > <plugin>里面的函数用来复用
>
> config文件新增内容
>
> > 添加auto_play_canvas模块，用来存储新显示面板的配置信息
> >
> > > is_scroll定义了音符显示器的显示方式
> > >
> > > num定义了一次显示多少个
> > >
> > > is_adaptive表示音符显示是否自适应窗口大小
>
> 图标更新为高清图标
>
> 优化了修改进度后显示内容的更新
